# Ingress

#### Etapas de deploy:

- Acessar a documentação do [kind](https://kind.sigs.k8s.io/docs/user/ingress/) para obter os manifestos necessários.
- Configuração do manifesto para criar o cluster com suporte ao ingress-controller.
- Direciona o acesso das portas `80` e `443` do `host` para o node `control-plane`.

~~~yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
# Control-plane
- role: control-plane
  kubeadmConfigPatches:
    - |
      kind: InitConfiguration
      nodeRegistration:
        kubeletExtraArgs:
          node-labels: "ingress-ready=true"
  extraPortMappings:
    - containerPort: 80
      hostPort: 80
      protocol: TCP
    - containerPort: 443
      hostPort: 443
      protocol: TCP

# Workers
- role: worker
- role: worker
- role: worker
~~~

Na seção de `configuração do Deployment` deve-se incluir a configuração a seguir, para garantir que o `ingress-controller` seja agendado no node do `control-plane`.
Dessa forma, o Ingress Controller só pode ser agendado no nó com esse `nome do host` ou com essa `label`.

~~~yaml
spec:
  template:
    spec:
      nodeSelector:
        ingress-ready: "true"
  # ou  kubernetes.io/hostname: "kind-control-plane"
~~~

Verificar se os pods ficam Ready antes de 1m30s.

~~~sh
kubectl wait --namespace ingress-nginx \
  --for=condition=ready pod \
  --selector=app.kubernetes.io/component=controller \
  --timeout=90s
~~~

Obtendo o endereço de acesso do ingress.

~~~sh
kubectl get ingress example-ingress -o jsonpath='{.status.loadBalancer.ingress[0].hostname}'
~~~
